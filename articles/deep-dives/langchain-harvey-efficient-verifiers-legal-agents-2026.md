# 法律 Agent 验证器设计：如何将成本降低 60-1000 倍

> **核心论点**：法律 Agent 的验证成本可以通过批量评估和廉价模型替代来大幅降低——DeepSeek 可在保持低误通过率的同时，将验证成本降低 60-1000 倍。

法律领域是 Agent最难征服的战场之一：文档跨度大、专业知识门槛高、输出正确性标准严格。验证一个法律 Agent 的工作质量，本身就是一个昂贵的过程。Harvey 与 LangChain 联合发布的 [LAB 基准](https://www.langchain.com/blog/designing-efficient-verifiers-for-legal-agents)揭示了一个关键问题：**如何在不牺牲准确性的前提下，将验证成本降低一到三个数量级？**

## 问题：验证成本是法律 Agent 的核心瓶颈

法律工作的验证涉及大量独立标准。以 LAB 基准中的任务为例，每个任务平均有 50+ 个独立标准需要逐一评估。使用 Opus 4.7 作为基准验证器对 40 个任务（共 2,348 个标准）进行评估时，每次调用的成本快速累积。

在 Agent 后训练（RL）中这个问题被进一步放大——每个任务需要多次 rollout，验证成本呈指数增长。如果验证太贵，团队要么减少实验次数，要么完全放弃 RL 后训练。

## 方法一：批量评估——减少 token 消耗

传统方式是逐条评估：每个标准一次 LLM 调用。批量评估则将完整评分表（rubric）一次发送给模型，让它同时判断所有标准。

实验结果显示，在准确率基本持平的前提下，批量评估将 token 消耗降低了约 10 倍。这意味着验证同一批数据，批量方法比逐条方法快一个数量级、成本也低一个数量级。

## 方法二：廉价模型替代——DeepSeek 的惊人表现

研究团队测试了 GPT-5.5、Sonnet 4.6、DeepSeek v4 Flash 和 Claude Haiku 4.5 作为验证器，以 Opus 4.7 逐条评估为基准进行对比。

关键发现：**DeepSeek 可在降低 60-1000 倍成本的同时，保持接近 Opus 的验证质量**。更重要的是，经过针对性 prompt 调优后，DeepSeek 的误通过率（false pass rate）从 10.7% 降至 9.5%——这对于法律领域至关重要，因为漏过错误比误报更危险。

## 方法三：Prompt 调优——针对误通过进行优化

DeepSeek 原始 prompt 的一个主要问题是：它对「与要求相关但未完全满足」的情况过于宽容。调优后的 prompt 要求模型将每个标准的每个部分明确分解为清单，并要求模型在信息不完全清晰时保持谨慎。这种方法在批量和逐条两种模式下都将误通过率降低了一个百分点以上。

## 核心结论

| 方法 | 成本降低 | 质量影响 |
|------|---------|---------|
| 批量评估 | ~10x | 基本持平 |
| 廉价模型（DeepSeek） | 60-1000x | 接近基准，误通过率可接受 |
| Prompt 调优 | 额外优化 | 误通过率 10.7% → 9.5% |

验证器不只用于评估，也用于后训练。在 RL 场景中，廉价且可靠的奖励信号意味着团队可以运行更多实验、审计更多 rollout、更快迭代。这是构建世界级法律 Agent 的关键基础设施。
