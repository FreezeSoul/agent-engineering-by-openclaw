# Codex 如何改变科学发现的游戏规则：一个天体物理学的案例研究

> 本文分析 OpenAI Codex 在天体物理学家 Chi-kwan Chan 研究中的工程实践，揭示 AI 辅助算法发现背后的「生成-验证」循环设计，以及这对 Agent 工程的意义。

## 核心命题

Codex 正在重新定义 AI 在科学研究中的角色——不是替代科学家做判断，而是成为算法发现的「加速器」：生成候选解，由人类科学家验证。**这本质上是一个最小化的 evaluator loop：Agent 生成假设，人类验证结论。**

---

## 背景：科学计算的长期瓶颈

黑洞周围等离子体的模拟，是天体物理学长期面临的难题。

等离子体是超高温的带电粒子物质。在密度较高的区域，电子与离子频繁碰撞，可以将等离子体简化为流体建模。但**在黑洞附近的高温弥散区域，粒子几乎不碰撞，而是沿磁力线螺旋运动**。

这意味着模拟必须追踪数万亿电子和离子的每一次螺旋——即使世界最快的超级计算机，也将大部分算力消耗在计算这些微小转动上，而非科学家真正关心的宏观物理行为。

> 「几十年来，这限制了我们对黑洞等离子体模拟的真实度。」—— Chi-kwan Chan

Chan 的解决思路是：改变数学上的粒子运动追踪方式，让计算机不再需要直接追踪每一次微小螺旋。但探索所有可能的数学方案——纯手工——需要「海量时间」。

这就是 Codex 的切入点。

---

## 工程机制：「生成-验证」循环

Chan 使用 Codex 的方式，本质上是一个**最小化的 evaluator loop**：

```
Codex（生成器）→ 候选算法建议 → Chan（验证器）→ 测试已知解 → 有效则采用
```

**关键设计点**：

### 1. AI 生成，人类验证

Chan 明确表示，他使用 Codex 来**提议并实现数值方案**，这些方案可以被检查、测试和物理理解。这与「AI 直接给出答案」的模式完全不同。

> 「Codex 生成了许多可能的方法——并非所有都正确。但这没关系。大多数科学想法都会失败。重要的是这些算法可以被测试。一旦找到有效的，就能解锁之前不可能的模拟。」

### 2. 可测试性是关键

Codex 生成的不是最终答案，而是**候选假设**。这与科学方法论完全一致：可证伪性是一切。

> 「我们不接受一个想法，不是因为它来自爱因斯坦、一个聪明的学生或 AI 模型。我们只在反复测试后才接受它。」

### 3. AI 扩展了探索空间

在没有 AI 的情况下，科学家只能手工探索有限数量的数学可能性。AI 将这个空间扩展了几个数量级，同时保持了人类专家的最终判断权。

---

## 为什么这不是普通的「AI 写代码」？

Chan 的案例与普通 AI Coding 有本质区别：

| 维度 | 普通 AI Coding | Chan × Codex |
|------|--------------|-------------|
| **目标** | 完成编程任务 | 发现新算法 |
| **输出** | 可运行的代码 | 候选算法方案 |
| **验证** | 运行时正确性 | 物理解释 + 数学一致性 |
| **反馈循环** | Bug 报告 | 与已知解对比验证 |
| **Agent 角色** | 执行者 | 假设生成器 |

**核心洞察**：当任务目标是「发现」而非「执行」时，evaluator loop 的设计必须从「正确性检查」转变为「有效性验证」——科学假设无法被证明「正确」，只能被「尚未证伪」。

---

## 对 Agent 工程的启示

Chan × Codex 的模式揭示了科学发现场景中 evaluator loop 的独特设计：

### 1. 验证标准是「物理合理性」，而非「代码正确性」

在科学计算中，代码通过了编译器只是第一步，真正的验证是「新算法是否能正确描述物理现象」。这需要领域专家作为 evaluator，而非自动化测试。

### 2. Human-in-the-loop 是永久设计，不是临时补丁

Chan 的工作流中，人类始终是最终判断者。AI 扩展了探索空间，但没有替代人类科学家做判断。这是科学场景中 AI 应用的正确姿态。

### 3. 失败是预期输出，而非异常

> 「大多数科学想法都会失败。」

当 evaluator loop 用于发现任务时，「大部分候选被拒绝」是正常状态，不是系统问题。评估器需要设计成**高效过滤无效候选**，而非「尽可能多地接受」。

### 4. 可解释性是工程约束

Chan 明确选择使用 Codex「能检查、测试和理解」的方案，而非「给出最终结论」的方案。这意味着在科学场景中，**可解释性是工程约束条件**，而非可选项。

---

## 结论：科学发现作为 Agent 工程的新场景

Codex 在天体物理学的应用，揭示了 AI Agent 在科学发现场景中的正确角色：

**不是替代科学家做判断，而是成为假设生成的加速器——扩展探索空间，由人类专家验证。**

这与 OpenAI Daybreak 的漏洞发现/修复场景形成有趣的对比：两者都是 evaluator loop，但验证标准截然不同——Daybreak 是「是否修复漏洞」，Chan × Codex 是「是否符合物理约束」。**场景不同，evaluator 设计也不同，但底层机制一致：生成器与验证器必须来自不同分布，且人类判断在关键节点介入。**

如果这项工作成功，新算法将允许科学家模拟黑洞周围数万亿粒子——这意味着 decades 的科学难题被解锁。
