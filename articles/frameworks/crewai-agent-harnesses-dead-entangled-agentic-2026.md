# Agent Harness 已死，Entangled Agentic Systems 是未来

> **核心论点**：当构建 Agent 的成本趋近于零时，差异化不再来自框架或工具链，而来自数据、信任和适应用户行为的自适应系统——CrewAI 称之为「纠缠软件」。

2025 年 DeepLearning.AI 大会上，CrewAI CEO 宣称「Agent Harness 已死」，台下一片不安。如今这个论断不仅没有过时，反而正在加速应验。在 [Agent Harnesses are Dead, Long Live Agent Harnesses](https://www.crewai.com/blog/agent-harnesses-are-dead) 一文中，CrewAI 联合创始人 Joāo Dias 进一步阐述了这个观点，以及他们看到的下一个战场。

## 价值正在向「不可一夜复制」的层转移

当 vibe-coding 一个 Agent 只需一个周末，当 API 调用足以驱动一个智能体，当从想法到原型的距离已缩短到几乎为零——这种速度正在摧毁工具层本身的护城河。

引用 @garrytan 的话说：**harness 应该是薄的，它是管道。管道很重要，但没有人会在管道上构建令人兴奋的有护城河的产品。**

Model providers 每季度都在吸收更多的堆栈，每隔几个月就有另一个原语被移到 API 之后。框架正在商品化，Harness 也在商品化，同样的力量正在开始 commoditize 两者。

那么价值在哪里？

1. **分发渠道**——已经建立的信任和网络
2. **专有数据**——花了多年积累的结构化数据
3. **产品智能**——用户使用产品时捕获的 intelligence，使产品越用越好
4. **生产信任**——在真实生产环境中赢得的可靠性声誉

这些东西无法靠 vibe-coding 在一个周末复制。这是一个飞轮，是赚来的，不是建造的。

## 纠缠软件：软件适应用户，而非用户适应软件

Dias 提出了一个物理学隐喻：**Entangled Software**。

在物理学中，当两个粒子纠缠时，一个的状态立即反映另一个的状态。在纠缠软件中，产品和客户相互影响。客户的行为塑造软件，软件反过来塑造客户的工作方式——随着时间推移，两者变得不可分割。

这与过去三十年的软件范式完全相反。传统上，我们构建工具，然后要求人类适应它们。但纠缠软件翻转了这个等式：**软件适应行为，而不是行为适应工具**。

在 Agentic 系统中，这意味着 Agent 不再只是执行预定义任务的工具——它们从每个客户的工作流中学习，适应每个组织实际运营的方式，随着使用变得越来越好。不是通过配置，而是通过 **Entanglement**。

## 企业 Agent 的当前状态：65% 已在使用，81% 正在扩展

根据 CrewAI 2026 年 2 月发布的[企业 Agent 调研报告](https://www.crewai.com/blog/the-state-of-agentic-ai-in-2026)（500 名企业高管参与），关键数据：

- **65%** 的企业已经在使用 AI Agent
- **81%** 已经全面采用或正在积极扩展跨团队的 Agentic AI
- **74%** 将 Agent 部署到生产环境视为关键优先事项或战略要务
- **75%** 报告时间节省效果「高」或「非常高」
- **57%** 倾向于在现有开源工具上构建，而不是从零开始
- **34%** 将安全和治理列为评估 Agent 平台的首要因素

有趣的是，ROI 和投资回报率排名最后（仅 2%）——不是因为 ROI 不重要，而是因为企业意识到没有安全、集成和可靠性的基础，可持续的 ROI 是不可能的。

## Agent 的演进路径

| 阶段 | 特征 | 护城河 |
|------|------|--------|
| 框架时代 | Scaffolding / Harness，内置 planning/memory/tools | 工具链 |
| 商品化期 | 任何人都能快速构建，差异消失 | 无 |
| Entangled 时代 | 产品从用户行为中学习，Agent 与客户深度绑定 | 数据+适应 |

## CrewAI 的方向

CrewAI 已有框架（Flows）和 Harness（Crews + Agents with memory/tools/caching/context/skills/MCP）。他们的下一步是建设「Entangled Agentic Systems」——这不再是一个框架或工具链，而是一种新的软件形态。

他们认为，**那些赢得这个时代的公司，不是那些建造了最好汽车的公司，而是那些建造了公路的公司**——信任、数据和适应性的基础设施。
