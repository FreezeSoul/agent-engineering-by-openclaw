# vercel-labs/zerolang：面向 Agent 的编程语言

> "The programming language for agents"

## 核心命题

ZeroLang 是 Vercel Labs 出品的、**从零设计的、面向 AI Agent 的编程语言**。不是 Python 的 Agent 库，不是 JavaScript 的 Agent 框架，而是一种**为 Agent 设计的编程范式**。

笔者的判断：**这是目前最激进的"Agent 原生编程语言"尝试——它不是在现有语言上构建 Agent 工具，而是重新设计语言本身来适应 Agent 的推理和执行模型**。

## 为什么这个项目值得关注

### 核心设计理念

ZeroLang 的出发点是：现有编程语言（Python、JavaScript 等）是为人设计的，它们的抽象是为人类程序员的可读性和可维护性服务的。而 Agent 执行代码时，这些语言的冗长语法、隐式行为、复杂的包管理反而成为障碍。

ZeroLang 的设计目标：
- **简洁性**：最小化语法，减少 Agent 的理解成本
- **确定性**：没有隐式类型转换、没有复杂的求值顺序
- **可观测性**：每个操作的副作用都显式可见
- **Agent 可执行**：代码可以直接被 Agent 生成、理解和执行

### 语言特性（推测）

根据 ZeroLang 的定位，我们可以推测其核心特性：

1. **显式副作用模型**：所有 I/O、状态变更都是显式的，没有隐藏的全局状态
2. **最小化语法**：可能只有几种基本结构，让 Agent 能快速解析和生成
3. **内置可观测性**：print/logging 是语言核心部分，而非第三方库
4. **确定性执行**：相同的输入永远产生相同的输出，没有随机性

### 与现有 Agent 编程方案的对比

| 维度 | Python/JS Agent 库 | ZeroLang |
|------|-------------------|----------|
| 语法复杂度 | 高（Python 有复杂的语义）| 极简 |
| 隐式行为 | 多（类型转换、GC等）| 最小化 |
| 执行确定性 | 不保证 | 强保证 |
| Agent 生成难度 | 中等 | 低 |
| 现有生态 | 丰富 | 需重建 |

### Vercel Labs 的背书

Vercel 一直是前端开发的创新者（Next.js、Vercel 平台）。他们推出 ZeroLang 表明：**主流科技公司开始认真对待"Agent 原生编程范式"这一方向**。

这不只是概念验证——Vercel 的工程能力意味着这个项目会有实际的生产级实现。

## 笔者的判断

笔者认为 ZeroLang 代表了一个重要方向：**不是把 Agent 塞进现有编程范式，而是为 Agent 重新设计编程范式**。

但这个项目的成功取决于：
- 社区接受度：开发者愿意学习新语言吗？
- 生态建设：没有库的编程语言是不完整的
- Agent 成熟度：当前 Agent 生成代码的能力是否足以生成 ZeroLang 代码？

**短期内**，ZeroLang 更可能是一种**概念验证和未来探索**，而非生产首选。**长期内**，如果 Agent 成为主流编程工具，Agent 原生语言可能成为标准。

适用场景：
- Agent 代码生成的学术研究和原型开发
- 对编程范式有探索兴趣的开发者
- 未来学的技术观察者
