# xbtlin/ai-berkshire：基于 Claude Code 的多 Agent 价值投资研究框架

> 本文属于 GitHub Trending 高价值项目推荐系列

## 核心命题

投资研究长期是「一个人 + 一个 AI」的场景——你问一句，AI 答一句，结论看起来全面，实际上没有立场。**xbtlin/ai-berkshire** 把这个问题反过来：让 4 个独立 Agent 同时从巴菲特、芒格、段永平、李录四个完全不同的视角研究同一支股票，最终输出带价格区间和分层建议的可决策报告，而非两面讨好的「分析」。

## 一、为什么这个项目值得关注

### 1.1 多 Agent 不是噱头，是工程设计

大多数多 Agent 项目把「多 Agent」当卖点，实际上是串行调用。ai-berkshire 的 `/investment-team` 是真正的并行研究架构：

```
/investment-team 启动 4 个独立 Agent 同时研究一家公司。
每个 Agent 各自搜索网络、交叉验证数据、独立给出结论。
Team Lead 再综合。
```

根据其 README，四大师视角各司其职：
- **段永平**（商业模式）：C2M 护城河、差异化、竞争壁垒 → 评分 3.7/5
- **巴菲特**（财务估值）：扣现金 PE、ROIC、现金流质量 → 评分 4.4/5
- **芒格**（逆向思考）：失败场景、概率加权 → 评分 3.5/5
- **李录**（长期确定性）：10 年维度、管理层文化 → 评分 2.0/5

巴菲特说「真便宜」，李录说「不确定就不买」——这种冲突本身就是投资决策的真实状态。多 Agent 架构把这四种判断同时拉出来，不是为了凑观点，而是让矛盾先暴露，再由人做最终决策。

### 1.2 实盘验证，不是 demo

这个项目最反常识的地方：作者直接公开了富途证券真实账户的截图。

|------|-----------|-----------|

| 标普 500 | +23.31% | +16.39% |

两年累计超额收益显著。README 明确标注「免责声明：历史收益不代表未来表现，截图来自富途证券真实账户」。

笔者认为：这不是在卖课程或卖策略付费内容，而是直接告诉你「我用这套框架实盘跑，资金量不大但确实在跑」。这种透明度在投资类 AI 项目里极为罕见。

### 1.3 四层防骗机制，解决的是 LLM 的系统性缺陷

AI 投资分析有两个常见问题：
1. **信息多 = 确定性高的幻觉**：研报读得越多越觉得自己懂
2. **LLM 心算不可靠**：PE 算错一个零、小数点搞混货币单位

ai-berkshire 的解决方式是把检验机制写进流程：

| 机制 | 解决什么问题 |
|------|-------------|
| 信息丰富度评级（A/B/C）| 防止「数据多 = 确定性高」幻觉，泡泡玛特评为 B 级，置信度明确标注 |
| 芒格式逆向检验 | 强制思考「什么情况下这笔投资会死」，列出 5 大情景及概率 |
| 快速否决清单 | 8 条红线一票否决，管理层诚信污点直接否决，不管估值多便宜 |
| 反共识检查 | 「聪明人为什么在做空？」→ 发现被忽视的空头逻辑 |
| 留白原则 | 数据不足时标注「灰色地带」，不用推测伪装确定性 |

README 里有一个真实案例：分析腾讯时，不同来源的市值数据出现了「港币亿」和「人民币亿」两种单位。框架的处理方式是：

```bash
python3 tools/financial_rigor.py verify-market-cap \
  --price 510 --shares 9.11e9 --reported 4.65e12 --currency HKD
# ✅ 验证通过, 偏差仅 0.08%
```

所有计算使用 Python `decimal.Decimal`，关键数据至少 2 个独立来源交叉验证。

## 二、技术架构

### 2.1 三层设计哲学

```
┌─────────────────────────────────────────────┐
│              Skill 层（16 个明确入口）          │
│  深度研究 / 财报分析 / 行业筛选 / 持仓管理 / 思维工具  │
├─────────────────────────────────────────────┤
│              Agent 层（4 × 独立并行）              │
│  商业模式 Agent / 财务估值 Agent / 行业竞争 Agent / 风险管理层 Agent  │
├─────────────────────────────────────────────┤
│              工具层（精确计算 + 实时检索 + 报告抽检）  │
└─────────────────────────────────────────────┘
```

Skill 层把「你要做什么」抽象成 16 个明确入口。Agent 层每个 skill 内部都是 4 个 Agent 并行。工具层保证数据严谨性可验证。

### 2.2 Claude Code 是如何被使用的

这个项目本质上是 **Claude Code 的 Skill 系统应用**。作者把投资大师的方法论编码成 `.md` Skill 文件，Claude Code 读取后作为 Agent 的系统指令执行。

关键 Skill 文件：
- `skills/investment-research.md` — 四大师综合深度分析
- `skills/investment-team.md` — 多 Agent 并行投研团队（核心）
- `skills/management-deep-dive.md` — 管理层纵深研究
- `skills/earnings-review.md` — 财报精读，只读原始财报不依赖二手研报

### 2.3 输出格式的工程化

普通 AI 输出的格式每次都不一样，今天分析腾讯有护城河评分，明天分析美团可能就忘了。ai-berkshire 确保「同样的输入 → 结构一致、深度一致的输出」：

```
公司    通过?    能力圈   好生意   护城河   管理层   安全边际   综合
茅台    ✅通过    ★★★★☆  ★★★★☆  ★★★★☆  ★★★☆☆   ★★★★☆   4.7
腾讯    ✅通过    ★★★★☆  ★★★★☆  ★★★★☆  ★★★★☆   ★★★★☆   4.7
英伟达  ✅有条件  ★★★★☆  ★★★★☆  ★★★★☆  ★★★★☆   ★★★☆☆   4.3
拼多多  ❓灰色    ★★★★☆  ★★★★☆  ★★★☆☆  ★★★☆☆   ★★★★★   3.8
```

这意味着 7 家公司可以横向对比，评分标准完全一致。同一家公司半年后重新分析，直接对比变化。

## 三、与 Claude Code 多 Agent 生态的关系

### 3.1 为什么是 Claude Code 而不是通用 Agent 框架

这个问题值得思考。LangChain、CrewAI、AutoGen 都有多 Agent 能力，但 ai-berkshire 选择了 Claude Code。

笔者认为原因有两层：

**第一层**：Claude Code 的 Skill 系统允许把投资大师的方法论直接编码为系统指令，不需要额外的框架抽象。每个 Skill 就是一个 `.md` 文件，Claude Code 原生解析，没有中间层。

**第二层**：Claude Code 的上下文窗口管理更适合长文本分析。投资研究需要读取财报、年报、多方观点，上下文质量直接影响分析质量。Claude Code 在长上下文场景下的指令遵循度更高。

### 3.2 多 Agent 并行 ≠ 简单 prompt 拆分

大多数人对「多 Agent」的理解是：把一个 prompt 拆成四段让四个 AI 各自跑。这个理解是错的。

ai-berkshire 的 4 个 Agent 各自做了完整的研究：
- 每个 Agent 独立搜索网络
- 每个 Agent 独立判断
- 每个 Agent 互相挑战（芒格 Agent 的逆向结论会挑战巴菲特 Agent 的乐观估值）
- Team Lead 最后综合

这不是 4x 的信息量，是 4x 的独立判断，最终通过结构化冲突得出更可靠的结论。

## 四、竞品对比

| 项目 | 特点 | 适用场景 |
|------|------|---------|
| **ai-berkshire** | 4 大师方法论 + 多 Agent 并行 + 实盘验证 | 股票投资研究，追求决策纪律 |
| **anthropics/financial-services** | Claude 官方金融 Agent 模板，Cowork/Managed Agents 双模式 | 企业级金融工作流 |
| **CrewAI** | 通用多 Agent 编排框架 | 需要自定义 Agent 角色的通用场景 |
| **MetaGPT** | 软件开发多 Agent，SOP 驱动 | 复杂代码生成任务 |

ai-berkshire 的差异化在于：它是**垂直领域的方法论编码**，不是通用多 Agent 框架。LangChain 能做多 Agent，但没有人把巴菲特的方法论写成 LangChain Skill。ai-berkshire 证明了 Claude Code Skill 系统在垂直领域方法论编码上的可行性。

## 五、如何快速上手

### 5.1 环境准备

```bash
# 安装 Claude Code（如果你还没有）
npm install -g @anthropic-ai/claude-code

# 或者使用 Codex（项目同时支持）
```

### 5.2 启动多 Agent 投研团队

```bash
# 进入 Claude Code，激活 investment-team skill
/investment-team

# 输入你想研究的公司，比如「拼多多」
```

Claude Code 会启动 4 个并行 Agent，分别从四个大师视角研究，完成后汇总报告。

### 5.3 快速否决检查

如果只是想做初步筛选：

```bash
# 启动财报精读
/earnings-review

# 或者管理层深挖
/management-deep-dive
```

## 六、笔者评价

### 这个项目解决了一个真实的问题

普通人直接问 AI 分析股票，AI 给你一篇「一方面...另一方面...」的平衡分析，最后以「投资有风险，请自行判断」收尾。这种分析看起来对，但没法拿来做决策。

ai-berkshire 解决的是**分析质量和决策纪律**的问题，不是分析能力的问题。它的核心假设是：投资研究需要多视角对抗，而多视角对抗需要多 Agent 并行架构，单一 AI 对话窗口无法模拟真实的投资决策过程。

笔者认为这个假设是成立的。多 Agent 框架的价值不在于「回答得快」，而在于「回答得不一样」——四个 Agent 给出的不同判断，制造了真实的矛盾和张力，而不是一个看似全面实则没有立场的综合答案。

### 工程层面的启示

对于 Agent 开发者来说，ai-berkshire 的意义在于：**它证明了把领域专家的方法论编码成 Skill 文件是可行的**。巴菲特的方法论被写成了结构化指令，Claude Code 读取后自动执行，不需要额外的推理引擎。

这意味着 AI Agent 的专业化路径可能不是「训练一个垂直领域的模型」，而是「把领域方法论编码为 Skill」。前者成本高，后者成本低，而且可以随时迭代更新。
