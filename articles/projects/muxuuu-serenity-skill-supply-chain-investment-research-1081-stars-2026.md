# muxuuu/serenity-skill：投研视角的供应链瓶颈Agent Skill

> **核心判断**：当大多数Agent Skills聚焦于「写代码、debug、refactor」等软件工程场景时，serenity-skill用一份MIT许可的开源实现告诉业界：**Skill模式可以走出软件工程，渗透到专业投研领域**——把热点拆解成产业链、把产业链映射到股票和基金、把模糊热度变成有结构化证据的研究方向。这是Skill从「工程师工具」到「分析师工具」的关键一步。

---

## 背景问题

2026年中，AI、半导体、机器人、CPO、算力等热点持续刷屏。大多数人遇到的问题是：

- **知道热，但不知道怎么切入** — 全网都在说AI半导体，但不知道该看哪条产业链、哪类公司、哪只基金
- **能感受热度，但分不清真假** — 哪些是真实供需瓶颈，哪些只是蹭热点
- **缺乏结构化研究流程** — 每天刷消息很焦虑，没有固定筛选流程

serenity-skill的解决思路来自@aleabitoreddit公开的投研方法论：

**从热点出发 → 拆产业链 → 找供应链瓶颈 → 筛候选公司和基金 → 检查公告/财报/风险 → 整理优先研究清单**

---

## 核心设计

### 1. 工作流程：六步研究法

```
热点 → 产业链拆解 → 瓶颈识别 → 标的筛选 → 证据核验 → 优先级排序
```

**第一步：热点拆解**
从大热点开始，先看真实需求来自哪里（不是看市场说什么，而是看产业链实际需要什么）。

**第二步：产业链拆解**
把主题分成下游需求、系统集成、芯片/器件、设备、材料、封测、基础设施等层级。

**第三步：瓶颈识别**
找低供应商数量、长验证周期、扩产困难、客户认证严格、材料纯度要求高的环节——这些才是真正难绕开的「卡点」。

**第四步：标的筛选**
回到股票和基金方向，判断谁更靠近真实瓶颈，谁主要只是蹭主题。

**第五步：证据核验**
检查公告、财报、问询函、订单、产能、客户和风险，给出有证据支撑的判断。

**第六步：优先级排序**
按产业链位置、证据强度、估值压力、风险点做优先级排序。

### 2. 证据 ladder：核验层级设计

serenity-skill内置了一套**证据层级体系**（evidence ladder）：

| 层级 | 证据类型 | 可信度 |
|------|---------|--------|
| L1 | 监管文件、交易所问询函 |最高 |
| L2 | 财报电话会披露 | 高 |
| L3 | 公告、招投标信息 | 中 |
| L4 | 客户/供应商公告 | 中 |
| L5 | 专利、标准文件 | 中低 |
| L6 | 可信媒体分析 | 低 |

**核验原则**：所有公司判断必须回到L1-L3级证据，不满足则降级。

### 3. 多客户端适配

支持多种Agent客户端的Skill目录结构：

```bash
# 用户级安装（Claude Code）
SKILL_DIR="$HOME/.claude/skills/serenity-skill"
mkdir -p "$SKILL_DIR"
cp -R SKILL.md LICENSE references assets scripts examples agents "$SKILL_DIR"/

# Agents级安装
SKILL_DIR="$HOME/.agents/skills/serenity-skill"
mkdir -p "$SKILL_DIR"
cp -R SKILL.md LICENSE references assets scripts examples agents "$SKILL_DIR"/

# Hermes级安装
SKILL_DIR="$HOME/.hermes/skills/research/serenity-skill"
```

## 使用示例

### 场景1：AI半导体热点扫描

```
用 serenity-skill 深度调研现在 A 股 AI 半导体产业链。
请联网查公告、财报、问询函、互动易、招投标、环评/能评、专利、客户认证和财务质量，
先排产业链层级，再找 5 个最值得优先研究的标的，
并说明卡住的环节、产业链位置、证据、排序理由和主要风险。
```

### 场景2：机器人产业链分析

```
用 serenity-skill 帮我研究最近机器人方向。
先拆产业链，再判断哪些环节更接近真实供需瓶颈，
最后给出股票和基金方向的优先研究清单。
```

### 场景3：标的挑战

```
用 serenity-skill 挑战 [公司/股票代码]。
它到底卡在哪一层？证据够不够？市场可能高估了什么？
什么情况说明这个判断应该降级？
```

## 战略意义

**1. Skill模式从「软件工程」到「专业投研」的边界突破**

目前主流的Agent Skills（如anthropic/skills、addyosmani/agent-skills）全部聚焦软件工程场景。serenity-skill证明了Skill范式可以泛化到**专业领域的结构化研究流程**——这对Skill生态的发展有重要的示范意义。

**2. 把@aleabitoreddit的方法论产品化**

Serenity（@aleabitoreddit）长期围绕AI、半导体、光通信、机器人等科技主题做供应链研究，核心思路是「大行情里真正有价值的机会，藏在系统扩张时最难绕开的关键环节」。这个Skill把这个思路做成了可复用的结构化工具。

**3. 与其他Skill形成互补**

| Skill | 聚焦领域 | 互补点 |
|------|---------|--------|
| anthropic/skills | 软件工程 | ✅互补 |
| addyosmani/agent-skills | 工程师技能 | ✅ 互补 |
| google-deepmind/science-skills | 科学研究 | ✅ 互补 |
| **serenity-skill** | **投研分析** | **新增维度** |

## 适用场景

**推荐使用**：
- 面对热点信息流，希望建立系统筛选流程
- 想学产业链研究方法论
- 对特定公司/行业做投研前的初步筛选
- 验证自己持有的股票是否「只是蹭热点」

**不推荐使用**：
- 需要最终买卖决策（Skill只做研究+排序）
- 对单一标的做深度尽调（需要更深入的实地调研）

## 笔者观点

serenity-skill最值得注意的不是它的技术实现，而是它的**定位选择**：它没有去卷软件工程Skills（那个赛道已经太拥挤），而是把Skill模式带到了一个完全不同但同样需要结构化流程的领域——**投研**。

这种「把领域专业知识的结构化流程做成Skill」的方法论，比Skill本身更有价值。如果你所在的领域也有类似的结构化研究流程，serenity-skill提供了一个很好的参考模板。
