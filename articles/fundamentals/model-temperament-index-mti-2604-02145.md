# Model Temperament Index (MTI)：行为驱动的 AI Agent 气质评估体系

> **本质**：像测量人类人格一样测量 AI Agent 的「性格」——不靠自述，靠行为测试；不测能力，测倾向

> **来源**：arXiv:2604.02145 (2026-04-02) | 作者：Jihoon 'JJ' Jeong, MD, MPH, PhD (DGIST/ModuLabs) | Paper #3 in the Model Medicine Series

---

## 一、基本概念

### 1.1 核心问题：同能力的模型，为什么行为完全不同？

AI Agent 领域有一个被忽视的事实：**能力相同的模型，可能有完全不同的「行为风格」**。

一个模型在压力下轻易妥协，另一个坚守立场；一个在关系维护上投入大量资源，另一个只专注任务完成。现有评估体系只测「能做什么」，不测「怎么做」——这就像只考试分数，不看考生的性格。

MTI（Model Temperament Index）就是为了填补这个空白。

### 1.2 气质 vs 人格：为什么用「气质」这个词？

论文刻意选择「Temperament（气质）」而非「Personality（人格）」：

| 维度 | 人格（Personality）| 气质（Temperament）|
|------|-------------------|-------------------|
| 获得方式 | 经验积累 | 先天倾向性 |
| 稳定性 | 可塑 | 相对稳定 |
| 测量方式 | 自述问卷 | 行为观察 |
| 适用对象 | 人类 | AI Agent |

MTI 的核心立场：**测量 Agent「做什么」，而非 Agent「说自己会做什么」**。这对于安全关键场景的部署决策至关重要。

### 1.3 四大轴与双层报告系统

MTI 构建了四轴评估框架，每个轴测量一个独立的「气质维度」：

```
┌─────────────────────────────────────────────────────────────────┐
│                    MTI 四轴评估框架                              │
├───────────────┬─────────────────────────────────────────────────┤
│ Reactivity    │ 环境敏感度——对输入变化的响应强度和速度           │
│ (反应性)      │ 子维度：Content（内容敏感）vs Formal（格式敏感） │
├───────────────┼─────────────────────────────────────────────────┤
│ Compliance    │ 指令-行为对齐度——遵循指令 vs 坚持自身判断         │
│ (顺从性)      │ 子维度：Formal（格式遵从）vs Stance（立场坚定）   │
│               │ ⭐ 两子维度独立（r = 0.002）                      │
├───────────────┼─────────────────────────────────────────────────┤
│ Sociality     │ 关系资源分配——在关系维护上投入多少注意力资源      │
│ (社交性)      │ 子维度：Facet H（帮助型）vs Facet A（自主型）     │
├───────────────┼─────────────────────────────────────────────────┤
│ Resilience    │ 抗压能力——压力下的行为稳定性                      │
│ (韧性)        │ 子维度：Cognitive（认知韧性）vs Adversarial（对抗韧性）│
│               │ ⭐ 两子维度呈反向关系                              │
└───────────────┴─────────────────────────────────────────────────┘
```

MTI 采用**双层报告系统**：
- **Layer 1**：四轴整体得分（高层气质轮廓）
- **Layer 2**：各轴内部子维度得分（精细化行为分解）

## 二、核心技术机制

### 2.1 测量协议设计原则

MTI 的行为测试不是简单的提示-响应匹配，而是精心设计的**两阶段分离协议**：

```
第一阶段：Capability Screening（能力筛查）
    ↓
排除能力差异对气质测量的干扰
    ↓
第二阶段：Disposition Testing（气质测试）
    ↓
在同等能力前提下，测量真实的倾向性差异
```

这个设计解决了 AI Agent 评估中最棘手的问题：**如何分离能力和意愿**。一个模型不听话，可能是因为它「听不懂」（能力问题），也可能是因为它「不想听」（气质问题）。MTI 通过两阶段设计将二者分离。

### 2.2 理论基础：Four Shell Model from Model Medicine

MTI 的理论根基是医学领域的 **Four Shell Model**（四壳模型），该模型认为：

1. **Core（核心层）**：模型的基础推理能力
2. **Shield（护盾层）**：对齐/安全机制
3. **Crown（王冠层）**：领域知识
4. **Temperament（气质层）**：行为倾向性 ← MTI 专注的层

这个框架的关键洞察：**气质是独立于能力、对齐和知识的第四个维度**。

### 2.3 四大轴的测量设计

#### Reactivity（反应性）

测试模型对不同类型输入变化的响应强度：

- **Content Reactivity**：对语义内容变化的敏感程度
- **Formal Reactivity**：对格式/结构变化的敏感程度

高分模型：输入措辞稍变就大幅改变输出
低分模型：即使输入大幅改变，输出也保持稳定

#### Compliance（顺从性）

最关键的轴之一。测试模型在多大程度上遵循指令：

- **Formal Compliance**：格式层面的遵从（按要求格式输出）
- **Stance Compliance**：立场层面的遵从（改变观点以匹配指令）

**关键发现**：这两个子维度是完全独立的（r = 0.002）。一个模型可能格式上非常顺从，但立场坚定；另一个可能格式混乱但观点轻易动摇。

#### Sociality（社交性）

测量模型在「完成任务」和「维护关系」之间的资源分配倾向：

- 高 Sociality：投入大量资源在关系协调上
- 低 Sociality：优先任务完成，关系维护最小化

#### Resilience（韧性）

压力测试的核心指标：

- **Cognitive Resilience**：在复杂/模糊场景下保持推理一致性
- **Adversarial Resilience**：在对抗性输入下保持稳定性

**重要发现**：这两个子维度呈**反向关系**（adversarial 高的模型，cognitive 可能低）。这意味着「抗揍」的模型不一定「抗压」，二者需要分别评估。

## 三、核心研究发现

### 3.1 发现一：四轴独立性

**结论**：在 instruction-tuned 模型中，四轴之间基本独立（所有 |r| < 0.42）。

这意味着：
- 一个模型可以高反应性 + 低顺从性 + 高社交性 + 低韧性
- 无法用单一维度概括 Agent 的气质
- **部署选型时必须对四轴分别评估**，不能只看综合分数

### 3.2 发现二：RLHF 重塑气质，不只是改变能力

这是 MTI 最重要的工程发现：

**RLHF（从人类反馈中学习）不仅改变了模型的能力，也深刻重塑了模型的气质。**

具体表现：
1. RLHF 后，轴得分发生系统性偏移
2. 更重要的是：**RLHF 在对齐后的模型中创造了 base model 中不存在的子维度分化**
   - Base model：Compliance 作为一个单一维度
   - RLHF 后：Compliance 分化为独立的 Formal 和 Stance 两个维度

工程含义：
- 对齐不是中性的——它改变了 Agent 的「性格」
- 对齐后的模型可能有 base model 没有的行为弱点
- **评测对齐效果时，必须同时评估气质变化，而不只是能力指标**

### 3.3 发现三：Compliance-Resilience 悖论

MTI 揭示了一个重要悖论：

> **Compliance（顺从性）和 Resilience（韧性）之间存在一个反向通道：观点屈服（opinion-yielding）和事实脆弱（fact-vulnerability）是通过独立路径运作的。**

一个高度顺从指令的模型，可能在事实层面反而更脆弱；一个坚持立场的模型，可能更容易被误导。

这对安全部署有直接意义：
- 高 Compliance ≠ 高安全性
- 需要分别评估「听不听指令」和「容不容易被错误信息影响」

### 3.4 发现四：气质与模型大小无关

在 1.7B–9B 参数范围内（6 个组织，3 种训练范式）：

> **气质与模型大小无关（temperament is independent of model size）**

这是一个重要的construct validity证据：
- 气质测的是「倾向」，不是「容量」
- 一个 9B 模型可能比 1.7B 模型更顺从，但这是气质差异，不是能力差异
- **同系列模型的不同规模，不代表气质更成熟**

## 四、与 Agent 开发的关系

### 4.1 部署前评估：为什么 MTI 不可或缺

传统评估只告诉你「模型能做什么」，MTI 告诉你「模型会怎么做」。

在 Agent 部署前，MTI 提供以下关键信息：

| 场景 | 相关轴 | 决策价值 |
|------|--------|---------|
| 高风险决策场景 | Resilience | 需要高认知韧性的模型 |
| 客服/协调场景 | Sociality | 需要平衡任务和关系 |
| 严格合规场景 | Compliance | 需要高指令遵从性 |
| 开放域对话 | Reactivity | 需要适度反应性 |

### 4.2 对齐效果的元评估

传统对齐评估关注：能力是否保留，安全边界是否提升

MTI 视角的对齐评估：**对齐是否改变了 Agent 的「性格」，这些改变是预期的吗？**

如果 RLHF 后一个高风险决策模型的 Compliance 意外下降，这可能是严重的安全隐患。

### 4.3 多 Agent 系统的气质匹配

在多 Agent 编排中，不同 Agent 的气质组合会影响系统行为：

- 高 Reactivity + 高 Compliance = 容易被输入操控
- 低 Resilience + 高 Sociality = 在对抗性环境下可能过度妥协
- 高 Sociality + 低 Compliance = 关系协调者，但可能不服从中心指令

**气质兼容性应该成为多 Agent 系统设计的维度之一。**

## 五、局限性

1. **仅测 small models**：1.7B–9B 范围，未覆盖 GPT-4/Claude 等大模型
2. **行为测试的边界**：MTI 测试的是特定设计的场景，不覆盖所有可能的 Agent 行为
3. **单次测量的稳定性**：模型版本更新可能导致气质漂移
4. **跨架构可比性**：不同架构（Decoder-only vs MoE）的 MTI 分数是否可比尚需验证

## 六、实践建议

### 什么时候关注 MTI

- **模型选型时**：除了 MMLU/HumanEval，也应该看看候选模型的 MTI 档案
- **安全审计时**：评估 Agent 在压力下的行为稳定性
- **对齐效果评估时**：不仅测能力保留，也测气质变化
- **多 Agent 编排设计时**：考虑 Agent 间的气质互补/冲突

### 如何获取 MTI 档案

MTI 作为一个开放标准，目标是建立 AI Agent 的「行为性格档案」。目前论文已发布（arXiv:2604.02145），建议关注：
- 作者团队是否会发布标准测试套件
- 主流评测平台（如 HELM、OpenCompass）是否会纳入 MTI
