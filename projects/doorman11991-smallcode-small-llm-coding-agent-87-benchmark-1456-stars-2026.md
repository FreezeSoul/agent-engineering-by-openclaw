# Doorman11991/smallcode：小模型编程 Agent，87% 基准分

> 与本文（Google DeepMind SIMA 2）形成互补：SIMA 2 使用 Gemini 大模型探索 3D 具身智能，smallcode 探索的是「小模型也能做高质量 coding Agent」的极端效率路线——两种路径代表了大模型 vs 小模型在 Agent 时代的不同工程选择。

---

## 一、项目定位：小模型 + 高效率

smallcode 挑战了一个流行观点：**做 AI coding Agent 必须用大模型**。

项目声称：在 4B 活跃参数下达到 87% 基准分，证明了经过精细优化的小模型可以在编程任务上接近大模型表现。

---

## 二、技术架构与关键指标

### 2.1 核心设计

smallcode 的优化策略：

| 优化维度 | 方法 |
|------|------|
| 模型规模 | 4B 活跃参数 |
| 训练数据 | 精选编程数据集 |
| 量化 | INT4/INT8 量化支持 |
| 推理优化 | vLLM 后端 |
| 基准表现 | 87% pass@1 |

### 2.2 适用场景

- **资源受限环境**：无法运行 70B+ 大模型的本地环境
- **低成本部署**：需要降低 API 调用成本的场景
- **边缘设备**：需要本地运行的 coding Agent

### 2.3 技术栈

- **推理引擎**：vLLM（高效 LLM 推理）
- **量化**：AWQ/GPTQ（模型压缩）
- **基准测试**：覆盖多种编程语言和任务类型

---

## 三、与 SIMA 2 的主题关联

SIMA 2 使用 Gemini 大模型（极大参数）探索具身智能的边界，smallcode 探索小模型（4B 参数）的效率极限。两者共同揭示了一个重要趋势：

| 维度 | SIMA 2 | smallcode |
|------|--------|-----------|
| 模型规模 | 超大（Gemini） | 小型（4B） |
| 目标 | 能力上限 | 效率优化 |
| 场景 | 3D 具身智能 | 通用编程 |
| 资源需求 | 高 | 低 |
| 核心价值 | 能力突破 | 成本控制 |

两者代表了大模型与小模型在 Agent 时代的两种工程选择：追求能力上限 vs 追求效率边界。

---

## 四、启示

smallcode 的实践表明：
1. **模型规模不是唯一壁垒**：优化方法、数据质量同样重要
2. **小模型 Agent 是可行的**：在特定场景下，4B 模型足够用
3. **效率与能力的权衡**：不同场景需要不同的模型策略
